# Plan: Fully Functional Boss Export (PDF + Excel)

## Goal
Make the Boss Export screen generate complete, professional PDF and Excel files containing **all relevant user data** for the selected year and month, with formulas and formatting that work correctly in spreadsheet applications.

---

## Current State Problems

### PDF (`buildPdfHtml`)
- Missing employee profile fields beyond `employeeName`, `jobTitle`, `department`: `email`, `onboarding status`, `app settings`, `hourRate` only shown in subtitle not as structured data
- No colleague/multi-user model present in current data, so focus is on the single-owner's full profile
- Summary footer manually pre-calculates totals instead of using structure that could be parsed
- No page metadata (generated by, export date, app version)

### Excel (`handleGenerateExcel`)
- **All values hardcoded with JS math** ŌĆö violates the xlsx skill rule: "Always use Excel formulas instead of calculating values"
- Only 2 sheets: "Summary" and "Session Details" ŌĆö missing profile/settings metadata
- Column widths set, but no font, no header fill, no bold, no borders, no number formatting, no freeze panes
- Summary totals (Total Hours, Sessions, Days Worked, Earnings) are pre-calculated JS values, not `=SUM()` / `=COUNTA()` / `=AVERAGE()` formulas
- No sheet for Employee Profile metadata

---

## Scope

Both outputs (PDF and Excel) must contain **all relevant data** for the selected period:

1. **Employee Profile Block** ŌĆö name, email, job title, department, onboarding status
2. **App Settings Block** ŌĆö theme, notifications setting
3. **Hour Rate** ŌĆö the configured hourly rate
4. **Period Summary** ŌĆö total hours, total sessions, days worked, estimated earnings
5. **Session Detail Table** ŌĆö every field on every session for that year/month where `checkOutTime !== null`, sorted newest-first
6. **Export Metadata** ŌĆö generated timestamp, app name/version, selected period

### 3. Shared Filter Logic

Both `buildPdfHtml` and `handleGenerateExcel` duplicate the same year/month filter. Extract to:

```typescript
const getFilteredSessions = () => {
  return appData.sessions
    .filter(s => {
      const d = new Date(s.checkInTime);
      return d.getFullYear().toString() === selectedYear
        && MONTHS[d.getMonth()] === selectedMonth
        && s.checkOutTime !== null;
    })
    .sort((a, b) => b.checkInTime - a.checkInTime);
};
```

Used by PDF, Excel, and the existing `monthStats` computation.

## Constraints / Risks

- **React Native xlsx library styling limits**: true cell-level formatting (font color, fill, borders) is not supported. Best we can do is column widths, row heights, merge ranges, and formula strings. If full professional styling is required, the next step would be server-side generation ŌĆö but the user asked for this to work from the app.
- **Openpyxl feature parity**: the skill guidelines describe practices that assume a Python backend. We are matching what is practical on-device with the available library, prioritizing **data completeness, formula usage where supported, and structural correctness** over unavailable styling features.
- **Excel formulas in xlsx JS**: formula strings written as cell values ARE preserved in the written file and ARE calculated when opened in Excel. This is the most impactful deliverable we can make without a backend.

---

## Out of Scope

- Multi-employee / colleague exports ŌĆö current data model is single-owner
- Server-side Excel generation ŌĆö would require a backend
- Charting / pivot tables in Excel ŌĆö not supported by current library in RN context
- Arabic RTL layout in PDF ŌĆö current PDF HTML is LTR; adding RTL would be a separate enhancement
